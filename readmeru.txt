Система управления сайтом"Блоголёт" это простой и легкий движок блога. Все подробности на официальном сайте проекта:
http://blogolet.ru/

Минимальные требование к системе: php версии не ниже 5, модуль апача rewrite. 

Установка. Если у вас есть шелл доступ к серверу, то рекомендую шелл скрипт, который скачивает и расставляет необходимые права перед установкой, скачать скрипт:
http://litepublisher.googlecode.com/svn/trunk/utils/setup.sh

Иначе  вам следует скопировать файлы и папки движка в корень домена, установить права для записи (0777) на папки backup, cache, data, files, plugins, themes, lib, lib/include, lib/install, lib/languages , на файлы в папке lib/languages установить права 0666 и после этого открыть адрес домена в браузере, после чего заполнить форму создания. Все!

Если вы хотите, чтобы блоголёт смог автоматически обновлять свои файлы с выходом новой версии, то вы должны установить права 0777 на папку lib включая все ее подпапки. Также необходимо установить права 0666на все файлы в папке lib включая все файлы в подпапках. Также установить права 0777 на папки в plugins, а на все файлы в папке plugins включая подпапки права 0666. Вы можете не менять права, но тогда вы должны будете самостоятельно заливать новые файлы на сервер.

После заливки файлов на сервер откройте в браузере адрес вашего сайта, например для blogolet.ru это будет 
http://blogolet.ru/

Если конфигурация устраивает, то будет запущен инсталятор блоголёта, где вы заполните простую форму с названием и E-Mail администратора. В самом начале стоит форма выбора языка. Если по каким то причинам инсталятор не смог определить язык установки, то смените язык, нажав кнопку "Change language".

Установка в подпапку домена ничем не отличается, за исключением того, что необходимо исправить файл .htaccess изменив строку 
RewriteRule . /index.php [L]

на 
RewriteRule . /subdir/index.php [L]

где subdir имя папки, куда устанавливается блог.

Установка нескольких блогов на одном сервере на разных доменах. Вы можете в корневом файле index.php определить пути к файлам, которые будут находится для всех доменов в одной папке. Для этого достаточно прописать соответствующие пути в массив $paths, указав для индекса 'home' путь к общей папке. В папке домена оставить только два файла .htaccess и index.php, а также две папки: themes - папку с темами, files - папка для файлов.

Замечание по безопасности. В корневом файле index.php желательно написать имя вашего домена, например для blogolet.ru это будет
$domain = 'blogolet.ru';

удалив последующие три строки, где определяется имя домена из http запроса, так как такое определение может нести потенциальную угрозу вашему сайту. Также для повышения безопасности отредактируйте строку с 
define('secret', 'любые символы, желательно уникальные'); - это повысит общую безопасность блога.

Для управления блогом зайдите в простую панель управления по специальному адресу:
http://yourdomain/admin/

Если вы потеряли пароль, то вы можете восстановить пароль, перейдя по ссылке
http://yourdomain/admin/passwordrecover/

где надо ввести ваш E-Mail администратора, накоторый будет выслан новый пароль к блогу, то есть старый пароль будет заменен новым случайно сгенерированным.

Если вы не можете нормально зайти в админку, то вероятно вам придется отредактировать .htaccess файл следующим образом: замените строку
RewriteRule .* - [E=HTTP_AUTHORIZATION:%{HTTP:Authorization},L]

на три других строки:
RewriteCond %{HTTP:Authorization} .*
  RewriteCond %{QUERY_STRING} !HTTP_AUTHORIZATION
  RewriteRule ^(.*)$ $1?HTTP_AUTHORIZATION=%{HTTP:Authorization} [L,QSA]

Замечание для установки на денвер: из за проблем с отправкой почты инсталятор блоголёта может подвисать, а также в других местах, где есть отправка писем, поэтому для нормальной работы блоголёта с последними версиями денвера необходимо отредактировать файл index.php в корневой папке, а именно добавить первую строку
define('debug', '');

и закоментировать следующую (желательно, но не обязательно):
//ob_start();

Для тех, кто хочет сделать редирект с адресами www можно добавитьв .htaccess следующие строки:
RewriteCond %{HTTP_HOST} ^www.site.ru$ [NC]
RewriteRule ^(.*)$ http://site.ru/$1 [R=301,L]
